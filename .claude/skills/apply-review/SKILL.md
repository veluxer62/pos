## User Input

```text
$ARGUMENTS
```

## 개요

`code-review` 스킬이 생성한 리뷰 리포트를 분석하여, 사용자가 동의한 항목만 코드에 반영한다.
반영 범위를 명확히 합의한 뒤 코드를 수정하고, 완료 후 lint 검증 및 커밋 훅을 실행한다.

이 스킬이 코드를 잘 고치려면 리뷰 리포트의 **파일 경로와 수정 방법**이 필요하다.
리포트 없이 막연하게 호출된 경우 먼저 `/code-review`를 안내한다.

---

## 실행 흐름

### 1단계: 리뷰 리포트 파악

현재 대화에서 가장 최근의 코드 리뷰 리포트를 찾는다.
리포트는 `# Code Review Report` 헤더와 Critical/Warning/Info 섹션으로 구성된다.

**사용자가 인자로 항목 ID를 제공한 경우** (`$ARGUMENTS`에 W-001, C-002 등이 포함된 경우):
→ 해당 ID 항목만 처리 대상으로 확정하고 3단계로 이동.

**리뷰 리포트를 대화에서 찾지 못한 경우**:
```
현재 대화에서 코드 리뷰 리포트를 찾지 못했습니다.
먼저 `/code-review`를 실행하거나, 리뷰 내용을 붙여넣어 주세요.
```

---

### 2단계: 심각도 수준 선택

리뷰 항목 전체를 표로 정리하여 출력한다:

```
## 리뷰 항목 요약

| ID    | 심각도   | 제목                     |
|-------|----------|--------------------------|
| C-001 | Critical | ...                      |
| W-001 | Warning  | ...                      |
| W-002 | Warning  | ...                      |
| I-001 | Info     | ...                      |

종합 판정: PASS / NEEDS WORK / FAIL

어느 수준까지 반영할까요?
  1. Critical만 반영
  2. Critical + Warning 반영  ← NEEDS WORK/FAIL 판정 시 권장
  3. 전체 반영 (Info 포함)
  4. 항목 직접 선택
```

사용자가 판정을 이미 언급했거나 인자로 범위를 줬다면 재질문 없이 진행한다.

---

### 3단계: 제외 항목 확인

선택된 범위의 항목을 보여준다:

```
다음 항목을 반영합니다:
  [C-001] 파일경로:라인 — 제목
  [W-001] 파일경로:라인 — 제목
  [W-002] 파일경로:라인 — 제목

제외할 항목 ID를 입력하세요. (없으면 "없음" 또는 Enter)
```

사용자 응답을 반영해 최종 반영 목록을 확정한다.

---

### 4단계: 코드 수정

확정된 항목을 **하나씩 순서대로** 처리한다.

각 항목 처리 절차:
1. 리뷰의 파일 경로와 수정 방법을 확인한다.
2. `Read`로 현재 파일 내용을 읽는다.
3. 리뷰가 지시한 범위만 수정한다. 관련 없는 코드는 건드리지 않는다.
4. 새 파일 생성이 필요한 항목(예: 테스트 파일 부재)도 이 단계에서 처리한다.
5. 수정 완료 후 한 줄 보고: `✓ [W-001] order_status.dart:12 — 파라미터명 statusName으로 수정`

수정 방법이 리뷰에 명시되지 않은 경우, 추측하지 말고 사용자에게 확인한 뒤 진행한다.

---

### 5단계: 검증

모든 수정 완료 후 lint를 실행한다:

```bash
export PATH="$PATH:$HOME/flutter/bin"
dart analyze
```

오류가 발생하면 원인을 분석하고 수정한 뒤 재검증한다.
테스트 파일이 생성된 경우 해당 테스트도 실행한다:

```bash
export PATH="$PATH:$HOME/flutter/bin"
flutter test [테스트파일경로]
```

---

### 6단계: 완료 보고

반영한 항목과 결과를 간략히 요약한다:

```
## 반영 완료

| 항목  | 내용                  | 결과 |
|-------|-----------------------|------|
| W-001 | 테스트 파일 생성      | ✓    |
| W-002 | 파라미터명 수정       | ✓    |

dart analyze: No issues found
```

## 주의사항

- 리뷰 항목 ID가 없는 자유 텍스트 리뷰는 항목 번호(1, 2, 3...)로 대체한다.
- Info 항목은 기본적으로 반영 대상에서 제외한다. 사용자가 명시적으로 요청할 때만 포함한다.
- 한 항목 수정이 다른 항목과 충돌할 경우 사용자에게 먼저 알린다.
- FAIL 판정 리포트는 Critical 항목을 반드시 먼저 처리하도록 안내한다.
- 수정은 리뷰가 지적한 파일과 범위에만 한정한다. 코드 전반 리팩터링은 이 스킬의 범위 밖이다.
